# Obsidian Weekly 2026-01-12：Obsidian v1.11 正式发布！终于可以自动检查插件更新啦

> [!Abstract]
> **统计时间**：2026-01-04 21:00 ~ 2026-01-12 21:00

## 官方资讯

### Obsidian v1.11 公共版现已发布

#### 桌面端

**注意：** 最新的安装程序启用了 Cookie 加密。如果你使用网页浏览器插件，我们建议升级到 [最新安装程序](http://obsidian.md/download)，以避免你的 Cookie 以明文形式被其他应用访问。

##### 改进

* 为设置部分添加了图标。
* 现在在文本和列表属性中支持 Markdown 链接。当目标文件被移动或重命名时，内部链接会自动更新。
* 每日笔记：可以从预定义的格式列表中选择每日笔记的格式。
* 尝试将文件重命名为包含不安全字符（如 `#` 或 `]`）只会收到警告。
* 导入文件时（例如通过拖放），Obsidian 现在会自动从文件名中去除无效字符。
* 在文件右键菜单中添加了“复制路径”子菜单。
* **macOS**：开关和滑块现在采用与其原生样式相匹配的风格。
* 新增设置项：**文件与链接 › 默认打开的文件**。可在“最后打开的文件”、“新建笔记”、“特定笔记”或“每日笔记”之间选择。如果你之前使用“启动时打开每日笔记”选项，请确保更新你的库以使用此新设置。
* 新增用于存储插件密钥的“钥匙串”设置部分。插件 API 即将推出。
* 当选中文本时，将 URL 粘贴到编辑器中将把选中的文本转换为使用该 URL 的 Markdown 链接（例如 `[选中的文本](粘贴的URL)`）。
* 新增设置：**社区插件 › 自动检查插件更新**。Obsidian 将在后台每 3 天或应用更新后检查一次插件更新。
* URI：`new`、`open` 和 `daily` URI 操作现在支持新的 `paneType` 参数。

| 参数 | 描述 |
| :--- | :--- |
| `paneType=tab` | 在新标签页中打开 |
| `paneType=split` | 在新标签组中打开 |
| `paneType=window` | 在弹出窗口中打开（仅限桌面端） |

* 新增了“从系统根目录复制当前文件路径”命令。

##### 已修复

* 参考链接不再允许在链接定义前有空格。
* 修复了在笔记间导航时，当属性列表出现/消失时有时会导致闪烁的问题。
* 修复了文本和列表属性中的外部链接（例如 `[链接](http://example.com)`）无法在浏览器中打开的问题。
* 修复了应用中“Inter”字体无法被识别的问题。
* 从右到左语言：修复了属性编辑器的布局问题。
* 图谱视图：附件不再显示完整文件路径。
* 数据基座：修复了配置分组时列表视图虚拟化的问题。修复了滚动时分组不显示的问题。
* 每日笔记：修复了当每日笔记格式包含文件夹路径时，“打开下一条每日笔记”和“打开上一条每日笔记”命令无效的问题。
* 修复了阅读模式中加粗链接（例如 `**[[链接]]**`）的显示问题。
* 使用“导出为 PDF”功能时，基础样式现在可以被完整导出。
* 修复了当文件名包含空格时，属性中的内部 Markdown 链接无法正常工作的问题。
* 修复了在主窗口和弹出窗口之间切换焦点可能导致错误的标签页被激活的问题。

##### 开发者相关

* 安装程序已更新至 Electron v39.2.6（需要下载 [最新安装程序](https://obsidian.md/)）。对于 Linux 用户，这默认启用了 Wayland 支持。
* Obsidian 现在捆绑了 Inter Variable 字体以替代 Inter。Inter Variable 支持可变字体字重。
* 向 API 添加了 `SettingGroup`。设置现在可以按逻辑分组，并可选地添加标题。这应为设置组织提供更标准化的设计语言。
* **API**：公开了新的 `SecretStorage` 和配套的 `SecretComponent`，以便插件与密钥存储集成。密钥存储是一种新的自愿加入方式，供插件保存可能在多个插件间共享的密钥（例如 Google API 令牌或 OpenAI 密钥）。现在插件可以从用户提供的名称标识的共享密钥集合中获取密钥，而无需用户将相同的密钥复制粘贴到多个插件设置中。

#### 移动端

##### 亮点新功能

###### IOS

* **锁屏小组件与控制中心小组件**：让你快速**新建笔记**、**打开特定笔记**、**打开每日笔记**、**打开搜索**或**打开 Obsidian**。
* **主屏幕小组件**：让你**新建笔记**、**查看笔记**或**打开每日笔记**。需要 iOS 18 或更高版本。当使用“需要面容 ID”来解锁应用时，小组件不可用。
* **快捷指令**：用于**打开笔记**、**打开每日笔记**、**捕捉到笔记**。捕捉快捷指令允许你向笔记追加或前置文本，而无需打开 Obsidian 应用。
* **Siri 集成**：例如 _“使用 Obsidian 捕捉”_, _“捕捉到 Obsidian”_, _“在 Obsidian 中打开我的每日笔记”_, _“在 Obsidian 中搜索”_。
* 在 Spotlight 中搜索 Obsidian 会显示额外操作：新建笔记、搜索和每日笔记。

###### Android

* **新小组件**：**打开笔记**、**新建笔记**、**搜索**、**每日笔记**、**打开 Obsidian**。你可以添加多个相同类型的小组件来打开不同的文件或触发不同的搜索查询。通过长按小组件进行配置。
* **快速设置磁贴**（需要 Android 7.0+）
* **新快捷方式**（需要 Android 7.1+）：打开笔记、每日笔记。通过长按应用图标、或将快捷方式图标拖动到主屏幕、或（对于大多数设备厂商）通过启动器的搜索栏访问。目前不可配置。

##### 改进

* 插件开发者现在可以请求用户位置，例如用于地图功能。
* 整个界面已焕然一新，以提升编辑和阅读体验。
* 在笔记中向下滚动时，导航栏会自动隐藏以最大化阅读屏幕空间。点击或稍微向上滚动可再次显示导航栏。
* 新的侧边栏导航，用于切换侧边栏标签页和执行操作，为更好的人体工学设计而对齐到底部。
* 切换到搜索视图时会立即聚焦到搜索输入框。
* 为开关和复选框添加了触觉反馈。
* 添加了双击将笔记从阅读模式切换到编辑模式的功能。
* 当使用每日笔记小组件时，如果尝试在未启用“每日笔记”核心插件的情况下打开每日笔记，将出现警告。
* Android 现在在可用时使用“Google Sans Flex”字体。
* 更新了我们的提示信息的设计。在手机上，提示信息现在将出现在屏幕顶部而非底部。
* 当导航栏隐藏时，减少了屏幕边缘的文本淡出效果。
* 在平板设备上，对话框现在具有更小的边框圆角。
* 当导航栏隐藏时，从视图顶部向下滑动可触发下拉操作。点击视图顶部可使导航栏重新出现。
* **iOS**：“查看笔记”小组件使用勾选标记渲染复选框清单。
* 为平板设备更新了侧边栏布局。标签页菜单和视图操作现在出现在顶部，以更好地模拟桌面端体验。
* 在文件管理器中长按文件现在会在单独的行上显示修改时间和创建时间。
* 文件菜单现在显示一个新的菜单项：**复制路径 › 从库文件夹**。

##### 已修复

* 修复了指定 `vault` 并指向 iCloud 库的 URI 会静默失败的 Bug。
* 如果你的库存储在 iCloud 中，我们会检查是否有任何配置文件需要从 iCloud 重新下载。文件下载完成后，你将看到一个提示，要求你重新加载应用。
* 修复了点击屏幕会将光标置于之前的光标位置，而非点击位置的 Bug。
* 在平板设备上，工具栏现在居中对齐。
* 改进了滑动手势的行为，当你开始滑动时内容将不再滚动（例如在设置中滑动返回）。
* 修复了键盘关闭时滚动位置会丢失的问题。
* **数据基座**：摘要行不再隐藏于导航栏之后。
* **同步**：修复了“防止睡眠”设置在重启应用后无法保存的问题。
* **iPad**：修复了窗口交通灯（关闭、最大化、最小化）出现在应用界面之上的问题。
* **iOS**：修复了运行 iOS 26.2+ 的设备在与元素交互时会导致工具提示出现（且永不消失）的问题。
